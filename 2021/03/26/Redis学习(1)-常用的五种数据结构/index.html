Redis学习(1)-常用的五种数据结构

Redis官方文档：http://www.redis.cn/commands.html

背景知识一、NoSQL 非关系型数据库 即 Not Only SQL

背景知识二、四类非关系型数据库

  1. Key-Value: Redis
  2. 文档型数据库(bson格式): MongoDB，一个基于分布式存储的数据库
  3. 面向列：HBase 分布式文件系统
  4. 图形化: Neo4J

1. Redis基础知识

redis 默认有16个数据库 默认使用的是第0个数据库 可以使用select index 切换数据库

Redis-benchmark: Redis自带性能测试服务

2. Docker 拉取镜像 创建并启动redis容器

3. Redis常用的五种数据结构

  1. key-string 一个key对应一个值
  2. key-hash 一个key对应一个map
  3. key-list 一个key对应一个列表
  4. key-set 一个key对应一个无序不重复集合
  5. key-zset 一个key对应一个有序不重复集合
  6. HyperLogLog 计算近似值
  7. GEO 存储地理位置
  8. BIT 存储一个Bit

4. Redis五种常用的数据结构图

String（字符串）

Hash（哈希）

Map集合 key-集合

存储变更的数据 user name age ,用户信息的保存 经常变动的信息

List（列表）

List实际上是一个链表，before node after left right都可以插入值

如果key不存在，则创建新的链表

如果key存在，新增内容

如果移除了所有的值，则成为空链表，也代表不存在

在两边插入或者改动值，效率最高，中间元素，相对来说，效率低一些

Set（集合）

无序集合，不能重复

应用：微博 A用户将所有关注的人放在一个set中 将所有粉丝放在一个set中，共同好友 可能认识的人 六度分割理论

ZSet（有序集合）

有序集合中的数字（score）代表权重

可按照权重排序实现输出

语法和set基本没有区别

应用：

  1. 存储班级成绩 工资表排序表
  2. 普通消息 带权重进行判断
  3. 排行榜应用实现
